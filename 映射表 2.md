| 优点生成规则 |  |  |

| 目标POI基础推荐分 | 方案内所有 POI 的平均推荐分 >= 85分 | 【高纯度量身定制】：行程内的景点/餐厅与您的个人偏好契合度极高，闭眼冲不踩雷。 |
| 交通消耗值 | 交通消耗值占总劳累度的比例 < 20% | 【高含金量行程】：通勤时间被压缩到极致，把宝贵的时间都留给景区，而不是浪费在路上。 |
| 行李搬运消耗值 | 行李搬运消耗值 = 0 （即换房次数为0） | 【行李零负担 / 一住到底】：全程无需打包行李、办理退换房，像本地人一样松弛度假。 |
| 景点游览消耗值 | 景点消耗值高，但用户标签是“特种兵/硬核打卡” | 【硬核充实，值回票价】：游玩时间拉满，专为精力充沛的您设计，绝对玩得尽兴。 |
| 劳累度（总和） | 每日总劳累度 <= 用户标签上限的 70% | 【高电量松弛感】：整体能耗极低，节奏温和，游玩结束后依然精力充沛，老少皆宜。 |

| 缺点生成规则 |  |  |
| 依赖的数据维度 | 触发条件 | 描述文案 |
| 目标POI基础推荐分 | 某必去 POI 的基础推荐分 < 60分 （用户硬要去的低分店） | 【局部体验可能踩雷】：行程中包含了个别与您标签匹配度较低的盲盒点，可能不符合您的预期。 |
| 交通消耗值 | 交通消耗值占总劳累度的比例 > 40% 或者是单日绝对值过高 | 【车乏感较重 / 路上时间长】：受限于地理距离，今日大半天在车上，建议自备颈枕、防晕车药。 |
| 行李搬运消耗值 | 行李搬运消耗值 >= 固定值 * 2 （换房两次以上） | 【中途频繁换房，略显折腾】：需要收拾多次行李并配合退改房时间，会碎裂掉一部分早晨的时间。 |
| 景点游览消耗值 | 景点消耗值过高，且用户标签是“带娃/陪长辈/躺平” | 【景点耗能过大，不宜硬撑】：部分景区游玩时间过长（如大范围步行/爬坡），长辈或宝宝容易电量耗尽。 |
| 劳累度（总和） | 每日总劳累度 > 用户标签上限 | 【特种兵强度，体力大考验】：今日总能耗爆表，微信步数可能破两万，晚上可能会有强烈的疲惫感。 |
